Conditional expressions which are always true or false can lead to dead code. Such code is always buggy and should never be used in production.

Noncompliant Code Example

a = false;
if (a) { // Noncompliant
  doSomething(); // never executed
}

if (!a || b) { // Noncompliant; "!a" is always "true", "b" is never evaluated
  doSomething();
} else {
  doSomethingElse(); // never executed
}

Exceptions

This rule will not raise an issue in either of these cases:

  • When the condition is a single const of boolean value 
const debug = false;
//...
if (debug) {
  // Print something
}
  • When the condition is the literal 0, 1, true or false.

In these cases it is obvious the code is as intended.

See

  • MITRE, CWE-570 - Expression is Always False
  • MITRE, CWE-571 - Expression is Always True
  • CERT, MSC12-C. - Detect and remove code that has no effect or is never executed

Deprecated

This rule is deprecated, and will eventually be removed.
