Sharing some naming conventions is a key point to make it possible for a team to efficiently collaborate. This rule allows to check that all variable, property and parameter names match one of the following patterns: lowerCamelCase, PascalCamelCase or UPPER_CASED. This rule also allows leading underscore.

Noncompliant Code Example

const foo_bar = 1;
const baz_ = 2;

Compliant Solution

const fooBar = 1;
const _baz = 2;{code}
