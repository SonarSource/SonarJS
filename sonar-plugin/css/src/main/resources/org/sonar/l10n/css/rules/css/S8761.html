Why is this an issue?

Many CSS shorthand properties support collapsed forms. For example, margin: 1px 1px is equivalent to margin: 1px, and padding: 1px 2px 1px is equivalent to padding: 1px 2px.

Keeping longer-than-necessary value lists adds noise, makes diffs harder to read, and can suggest that the values intentionally differ when they do not.

This rule raises an issue when a shorthand property can be written with fewer values without changing the rendered result. It applies to edge shorthands such as margin, padding, border-color, border-style, border-width, inset, scroll-margin, and scroll-padding, as well as border-radius, gap, overflow, overscroll-behavior, and their logical or vendor-prefixed variants.

How to fix it

Rewrite the declaration using the shortest equivalent shorthand form.

Noncompliant code example

a {
  margin: 1px 1px;
  padding: 1px 2px 1px;
  border-radius: 1px 2px 1px 2px;
  -webkit-border-radius: 1px 1px 1px 1px;
}

Compliant solution

a {
  margin: 1px;
  padding: 1px 2px;
  border-radius: 1px 2px;
  -webkit-border-radius: 1px;
  margin: 1px 1px 1px 2px;
}
