Why is this an issue?

In Node.js importing modules is doable by providing an absolute path such as /lib/foo/bar.js. Doing this restricts the portability of your code, making it specific to your computerâ€™s file system and potentially causing issues when the code is distributed, for example, through NPM packages.

How to fix it

Replace the absolute path with one that is relative to your current file.

Code examples

Noncompliant code example

import { foo } from '/home/project/api/bar.js';

Compliant solution

import { foo } from '../api/bar.js';
